================================================================================
    SISTEMA ITSI - GESTIÓN INTEGRAL DE BIENESTAR INSTITUCIONAL (GIBI-ITSI)
    RESUMEN DE FUNCIONALIDADES Y ROLES DEL SISTEMA

Tecnología: CodeIgniter 4 (PHP) | Base de datos: MySQL (bienestar_estudiantil_db)
Arquitectura: MVC (Modelo-Vista-Controlador)

================================================================================
                              ROLES DEL SISTEMA
================================================================================

El sistema maneja 3 roles principales, identificados por rol_id:

┌──────────────────────────────────────────────────────────────────────────────┐
│ ROL 1: ESTUDIANTE (rol_id = 1)                                             │
│ Dashboard: /estudiante                                                     │
│ Controlador: EstudianteController                                          │
│ Descripción: Usuario final del sistema. Puede gestionar su ficha           │
│ socioeconómica, solicitar becas, enviar solicitudes de ayuda y             │
│ administrar su perfil y cuenta personal.                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│ ROL 2: ADMINISTRADOR DE BIENESTAR (rol_id = 2)                             │
│ Dashboard: /admin-bienestar                                                │
│ Controlador: AdminBienestarController                                      │
│ Descripción: Administrador del departamento de bienestar estudiantil.      │
│ Gestiona fichas socioeconómicas, becas, períodos académicos, usuarios,     │
│ solicitudes de ayuda, reportes y configuración del sistema.                │
├──────────────────────────────────────────────────────────────────────────────┤
│ ROL 4: SUPER ADMINISTRADOR / GLOBAL ADMIN (rol_id = 4)                     │
│ Dashboard: /global-admin/dashboard                                         │
│ Controlador: GlobalAdminController                                         │
│ Descripción: Administrador global del sistema con control total. Gestiona  │
│ usuarios, roles, configuración del sistema, respaldos de base de datos,    │
│ logs de actividad y estadísticas globales.                                 │
└──────────────────────────────────────────────────────────────────────────────┘

Nota: El rol_id = 3 (Administrativo) existe en el mapa de roles pero no
tiene controlador ni rutas activas en el sistema actual.

================================================================================
                        MÓDULO DE SEGURIDAD
================================================================================

El sistema cuenta con un módulo de seguridad dedicado (app/Security/) que
incluye los siguientes componentes:

1. RateLimiter       - Control de frecuencia de peticiones para prevenir
                       ataques de fuerza bruta
2. InputSanitizer    - Sanitización y validación de datos de entrada
3. SecurityLogger    - Registro de eventos de seguridad (login, logout,
                       accesos denegados, bloqueos)
4. SessionGuard      - Protección avanzada de sesiones (inicialización,
                       destrucción segura)
5. CsrfGuard         - Protección contra ataques CSRF
6. XssProtector      - Protección contra inyección XSS
7. SecurityHelper    - Utilidades de seguridad (obtener IP, enmascarar emails)
8. PasswordPolicy    - Políticas de contraseñas seguras

Filtros de seguridad (app/Filters/):
- AuthFilter         - Verificación de autenticación en rutas protegidas
- RoleFilter         - Autorización basada en roles por ruta
- RateLimitFilter    - Filtro global de rate limiting
- SecurityHeaders    - Cabeceras HTTP de seguridad
- XssFilter          - Filtrado global contra XSS
